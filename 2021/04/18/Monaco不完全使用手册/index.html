Monaco不完全使用手册
2021/04/18 编辑器

Monaco Editor是为VS Code提供支持的代码编辑器，目前很多浏览器上的编辑器”都是基于 monaco-editor 来做的。这里以monaco editor的react版本为例，介绍Monaco的一些基本使用。

官网：https://microsoft.github.io/monaco-editor

1、安装

npm install react-monaco-editor
npm install monaco-editor-webpack-plugin // 依赖插件

插件安装好后修改修改 webpack配置文件，注入新的 plugin：

const MonacoWebpackPlugin = require('monaco-editor-webpack-plugin');

module.exports = {
  plugins: [
  // 根据项目实际需要自行配置
    new MonacoWebpackPlugin({
        filename: '[name].worker.[contenthash].js',
        languages: ['sql', 'json', 'javascript'],
        features: [
          'comment',
          'format',
          'fontZoom',
          'folding',
          'codelens',
          'suggest',
          'wordOperations',
          'wordPartOperations',
          'wordHighlighter',
          'parameterHints',
          'bracketMatching',
          'find',
          'linesOperations', // cmd + del 等快捷键操作
        ],
      }),
  ]
};

2、基本使用

import MonacoEditor from 'react-monaco-editor'

import * as monacoEditor from "monaco-editor/esm/vs/editor/editor.api"

const App: React.FunctionComponent = (props: AppProps) => {
  const handleChange = () => {
  }

  const editorDidMount = (editor: monacoEditor.editor.IStandaloneCodeEditor, monaco: typeof monacoEditor) => {
    editor.focus()
    // 常用API在下面列出
  }

  return (
    <>
      <MonacoEditor
        language="sql"
        editorDidMount={editorDidMount}
        onChange={handleChange}
      />
    </>
  )
}

这样就可以构建一个最简单的编辑器啦~

editor实例常用的API：

// editor为editorDidMount中获取到的实例
editor.getValue();           // 获取编辑器当前文本内容
editor.setValue();           // 设置编辑器文本内容
editor.getSelection();       // 获取选中的行信息
editor.getModel().getLineContent(1);      // 获取某一行信息
editor.getModel().getLinesContent();      // 获取每一行信息
editor.onDidChangeModelContent(e => {     // 内容改变事件
    e.changes[0].rangeOffset;     // 光标位置
    e.changes[0].text;            // 输入的文本
    ...
});

editor.addCommand(monaco.KeyMod.CtrlCmd | monaco.KeyCode.KEY_S, () => {  
    // 添加按键监听
    monaco.KeyMod  // 特殊键 ctrl alt shift win
    monaco.KeyCode  // 键盘其他键
});

editor.editor.setTheme('vs-dark');          // 设置主题

3、主题

monaco支持的原生主题有vs、vs-dark等：

<MonacoEditor
  {...props}
  theme='vs-dark'
/>

也可以自己基于原生主题进行定制：

import * as monaco from 'monaco-editor/esm/vs/editor/editor.api'

const BG        = '#252A33'
const ORANGE    = '#FFB86C'
const RED       = '#FF5555'
const BLUE      = '#1E90FF'

monaco.editor.defineTheme('vs-moonlight', {
    base: 'vs-dark',
    inherit: true,
    rules: [],
    colors: {
      'editor.background': BG,
    },
  })

4、自定义高亮

Monarch 是 Monaco Editor 自带的一个语法高亮库，通过它，我们可以用类似 Json 的语法来实现自定义语言的语法高亮功能。

4-1、基本语法

如果是自定义的语言需要先注册自定义语言名称，再设置高亮规则；如果是已有语言高亮则不需要第一步直接设置高亮规则就可以。

// 注册自定义语言名称
monaco.languages.register({ id: 'myLanguages' })
// 设置高亮规则
monaco.languages.setMonarchTokensProvider(
  'myLanguages', 
  monarchObj
)

4-2、 Monarch

Monarch 由一系列 Json 键值对组成，他有许多属性，其中最重要的就是 tokenizer 属性，我们描述语法的代码就写在这里面。

4-2-1、 rule-action

来看一个简单的栗子：

  monaco.editor.defineTheme('vs-moonlight', {
    base: 'vs-dark',
    inherit: true,
    rules: [
      ...
      { token: 'custom-info', foreground: BLUE },
      { token: 'custom-error', foreground: RED, fontStyle: 'bold' },
      { token: 'custom-warning', foreground: ORANGE }
    ],
    colors: {
      'editor.background': BG,
    },
  })

monaco.languages.setMonarchTokensProvider('myLanguages', {
    tokenizer: {
      root: [
        [/^[error]/, { token: "custom-error" }],
        [/^[info]/, { token: "custom-info" }],
        [/^[warning]/, { token: "custom-warning" }]
      ]
    },
  })

我们在 tokenizer 中定义了一个 root 属性，root 是 tokenizer 中的一个 state , 这就是我们用来编写解析规则（rule）的地方，在 rule 中，我们可以编写匹配文本的正则表达式，然后再给匹配到的文本设置一个执行动作的 action ，在 action 中，我们可以给匹配到的文本设置 token class 。

在上面的栗子中，分别写了三个rule，用来匹配[error]、[info]、[warning]，匹配成功后就接着执行对应的 action ；在 action 中，我们设置了匹配文本的 token class : custom-error、custom-info和custom-warning。

4-2-2、 token class

本质上，token class 其实就是设置 css 中的 class ，不过Monarch 中转换成一层对应关系，查看elements可以看到：

error对应class为mtk9，warning对应class为mtk11，info对应class为mtk10。

Monarch 中内置了以下几种 token class：

identifier         entity           constructor
operators          tag              namespace
keyword            info-token       type
string             warn-token       predefined
string.escape      error-token      invalid
comment            debug-token
comment.doc        regexp
constant           attribute
delimiter .[curly,square,parenthesis,angle,array,bracket]
number    .[hex,octal,binary,float]
variable  .[name,value]
meta      .[content]

例如keyword：

monaco.languages.setMonarchTokensProvider('myLanguages', {
    keywords: [
      'for', 'new', 'switch', 'goto', 'do', 'if', 'private', 'this', 'break', 'protected', 'throw', 'else', 'public',
      'return', 'catch', 'try', 'interface', 'static', 'class', 'finally', 'const', 'super', 'while'
    ],
    tokenizer: {
      root: [
        [/[a-z_$][\w$]*/, { 
          cases: { 
            '@keywords': 'keyword',
          }
        }],
      ],
    },
  })

被列入keywords中的文本将作为关键字高亮展示。

4-2-3、 cases-guard

在上面栗子中，我们一共写了三个rule，分别用来匹配[error]、[info]、[warning]，但这些rule其实很相似，因此可以将他们进行合并：

monaco.languages.setMonarchTokensProvider('myLanguages', {
    tokenizer: {
      root: [
        [/^[(\w+)]/, { token: "custom-$1" }],
      ],
    },
})

这里用一个美元符号$来代表取正则表达式第几个匹配项，$0代表取所有的匹配项（例：[error]），$1代表取第一个匹配项（例：error）。

上述代码将类型（error/info/warning等）作为参数传入了token class，与 custom-做拼接，从而组成了最终的token class，例如custom-error。

但如果新增其他类型（比如debug），但此时不希望匹配到debug，则需要利用cases做一些处理。

cases和普通的if /else if语法一样，可以写多个判断条件（guard），然后根据不同guard去执行对应的action。

guard和正则表达式类似，功能是用来匹配文本，当他不以@或$开头时，他就是一个普通的正则表达式，不过，当他以@或$开头时，他才是一个真正意义上的 guard。

语法：

{ cases: { guard1: action1, ..., guardN: actionN } }

guard有固定的结构 [pat][op]match，pat代表匹配的文本，op代表一个比较符，match则是要比较的内容。

pat 以$开头，和我们上文正则表达式使用的$1含义是一样的，不过这边$#代表全部匹配文本，而正则表达式是使用$0代表全部匹配文本。另外，我们还可以用 $Sn来获取当前state的名字，例如在root state下$S0就代表root。

op 和 match 稍微复杂点，可以是这几个内容：

  • regex or !regex ：匹配/不匹配一个正则
  • @attribute or !@attribute ：匹配/不匹配一个属性，属性定义在 Monarch 的根层级下，可以是数组、字符串、正则。
  • ==str or !=str ：匹配/不匹配一个字符串
  • @default ：匹配默认情况
  • @eos ： 一行结束，则匹配成功

因此，之前的代码可以简化为：

{
    keywords: ['error', 'info', 'warning'],
    tokenizer: {
        root: [
            [/^[(\w+)]/, {
                cases: {
                    "$1@keywords": { token: 'custom-$1' },
                    "@default": { token: "string" }
                }
            }]
        ]
    }
}

有了cases-guard，我们就可以随心所欲的定制自己需要的高亮规则啦：

  monaco.editor.defineTheme('vs-moonlight', {
    base: 'vs-dark',
    inherit: true,
    rules: [
      { token: 'shanshan', foreground: PINK }
    ],
    colors: {...},
  })

  monaco.languages.register({ id: 'myLanguages' })
}

monaco.languages.setMonarchTokensProvider('myLanguages', {
    customFields: ['shanshan', 'hello', 'world'],
    tokenizer: {
      root: [
          [/[\w$]+/, {
            cases: {
              '@customFields': 'shanshan'
            }
          }]
      ]
    }
  })

5、自动联想补齐

联想补齐主要利用monaco提供的registerCompletionItemProvider API，以SQL为例：

首先给出SQL语言格式的配置JSON：
点击下载 sql.ts

import { language } from './sql'

monaco.languages.registerCompletionItemProvider('myLanguages', {
    // triggerCharacters: ['.'], //触发智能提示关键词
    provideCompletionItems: (model, position) => {
      const textUntilPosition = model.getValueInRange({
        startLineNumber: position.lineNumber,
        startColumn: 1,
        endLineNumber: position.lineNumber,
        endColumn: position.column
      })

      const match = textUntilPosition.match(/(\S+)$/)

      const suggestions: monaco.languages.CompletionItem[] = []

      if (match) {
        const matchStr = match[0].toUpperCase();
        language.keywords.forEach((item: string) => {
          if (item.startsWith(matchStr)) {
            suggestions.push({
              label: item,
              kind: monaco.languages.CompletionItemKind.Keyword,
              insertText: item
            } as monaco.languages.CompletionItem)
          }
        })

        language.operators.forEach((item: string) => {
          if (item.startsWith(matchStr)) {
            suggestions.push({
              label: item,
              kind: monaco.languages.CompletionItemKind.Operator,
              insertText: item
            } as monaco.languages.CompletionItem)
          }
        })

        language.builtinFunctions.forEach((item: string) => {
          if (item.startsWith(matchStr)) {
            suggestions.push({
              label: item,
              kind: monaco.languages.CompletionItemKind.Function,
              insertText: item
            } as monaco.languages.CompletionItem)
          }
        })
      }

      return {
        suggestions: Array.from(new Set(suggestions))
      }
    },
  })

这样就可以实现SQL语句的自动联想补齐啦～

如果还需要进一步实现SQL关键字高亮，则只需要改动以下内容：

import { language } from './sql'

monaco.languages.setMonarchTokensProvider('myLanguages', language as monaco.languages.IMonarchLanguage)

6、格式化

仍然以SQL为例：

import sqlFormatter from 'sql-formatter';

// 格式化 SQL
monaco.languages.registerDocumentFormattingEditProvider('sql', {
  provideDocumentFormattingEdits(model) {
    const formatted = sqlFormatter.format(model.getValue());
    return [{
      range: model.getFullModelRange(),
      text: formatted
    }];
  }
});

7、插入文本

如果需要将指定文本插入当前光标位置，或用指定文本代替当前选中文本，实现如下：

  const insertText = value => {
    // editor为editorDidMount中获取到的实例
    const selection = editor.getSelection()
    const range = new monaco.Range(
      selection.startLineNumber,
      selection.startColumn,
      selection.endLineNumber,
      selection.endColumn,
    )

    const id = { major: 1, minor: 1 }

    const op = {
      identifier: id,
      range,
      text: value,
      forceMoveMarkers: true,
    }

    editor.executeEdits('my-source', [op])
    editor.focus()

  }
