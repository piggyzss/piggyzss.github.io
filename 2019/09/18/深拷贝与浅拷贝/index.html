深拷贝与浅拷贝
2019/09/18 code 前端

如果给一个变量赋值一个对象，那么两者的值会是同一个引用，其中一方改变，另一方也会相应改变，可以使用浅拷贝来解决这个问题。

1、浅拷贝方式

Object.assign

let b = Object.assign({}, a)

通过展开运算符（…）

通常浅拷贝就能解决大部分问题了，但是当我们遇到如下情况就需要使用到深拷贝了

let a = {
    age: 1,
    jobs: {
        first: 'FE'
    }
}
let b = {...a}
a.jobs.first = 'native'
console.log(b.jobs.first)

浅拷贝只解决了第一层的问题，如果接下去的值中还有对象的话，那么就又回到刚开始的话题了，两者享有相同的引用。要解决这个问题，我们需要引入深拷贝。

2、深拷贝方式

JSON.parse(JSON.stringify(object))

缺点：

  • 会忽略 undefined
  • 会忽略 symbol
  • 不能序列化函数
  • 不能解决循环引用的对象
  • 当然如果你的数据中含有以上几种情况下，可以使用 lodash 的深拷贝函数

3、函数解决

浅拷贝

function shallowCopy(src) {
  var dst = {}
  for(key in src){
    if(src.hasOwnProperty(key)){
      dst[key] = src[key]
    }
  }
  return dst
}
浅拷贝只会对对象的各个属性依次赋值，因此会导致src.obj和dst.obj指向同一块内存地址

深拷贝

function deepCopy(src){
  if (!src | !(src instanceof Object) | (typeof src == "function")) {
    return src||undefined;
  }
  var constructor = src.constructor;
  var dst = new constructor();
  for (var key in src) {
    if (src.hasOwnProperty(key)) {
      dst[key] = deepCopy(src[key]);
    }
  }
  return dst;
}

tips: 巩固知识点–数据存储方式

基本数据类型：number，string，boolean，null，undefined，symbol

通过栈内存保存
栈

变量名name 值value
a 1

当你b=a复制时，栈内存会新开辟一个内存，例如这样：

变量名name 值value
a 1
b 1

引用数据类型：object，array，function

栈内存保存地址指针，堆内存保存数值
栈

变量名name 值value
a 堆地址1

堆

堆
{name: ‘lily’, age: 12}

当b=a进行拷贝时，其实复制的是a的引用地址，而并非堆里面的值。
