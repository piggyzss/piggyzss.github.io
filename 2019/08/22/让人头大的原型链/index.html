让人头大的原型链
2019/08/22 code 前端

一、prototype & proto

1、几个概念

  • 构造函数：用来初始化新建对象的函数
  • 实例对象：通过构造函数的new操作符创建的对象

2、prototype

prototype是一个显式原型属性，只有函数才拥有该属性。
每个函数都有 prototype 属性(除了 Function.prototype.bind())，这个属性是一个指针，指向一个对象，这个对象包含可以由特定类型的所有实例共享的属性和方法。

当声明一个函数时，这个属性就自动被创建了

function Fn() {}

这个prototype属性的值时一个对象，在没有认为往prototype上挂载其他方法时，该属性只有一个值constructor(指向这个函数本身)

Fn.prototype // {constructor: ƒ}
Fn.prototype.constructor === Fn // true

此时，我们可以通过这样的方式对原型对象做扩展：
demo1

Fn.prototype.getInfo = function () {
  ...
}

3、proto

proto是隐式原型属性，指向了创建该对象的构造函数的原型。
每个对象都有proto属性，指向了创建该对象的构造函数的原型(也就是构造函数的prototype)。

因为在JS中是没有类的概念的，为了实现类似继承的方式，通过proto将对象和原型联系起来组成原型链，得以让对象可以访问到不属于自己的属性。

实例对象中的proto

当使用new操作符时，生成的实例对象拥有了proto属性
tips：new做了什么

  • 新生成了一个对象
  • 链接到原型
  • 绑定 this
  • 返回新对象
function create() {
    // 创建一个空的对象
    let obj = new Object()
    // 获得构造函数
    let Constructor = [].shift.call(arguments)
    // 链接到原型
    obj.__proto__ = Constructor.prototype
    // 绑定 this，执行构造函数
    let result = Constructor.apply(obj, arguments)
    // 确保 new 出来的是个对象
    return typeof result === 'object' ? result : obj
}

4、原型链

js中每个函数对象都有一个prototype属性（显式原型属性），称之为原型
而原型的值也是一个对象，因此它也有自己的原型（proto），这样就串联起一条原型链，原型链的链头是object，它的prototype比较特殊，值为null

function Fn() {}
var f = new Fn()
f.__proto__ === Fn.prototype // true
Fn.prototype.__proto__ === Object.prototype // true
Object.prototype.__proto__ // null 已经到原型链的链头了

原型链的作用是用来作对象继承，当一个函数被用作构造函数来创建实例时，该函数的原型属性将被作为原型赋值给所有对象实例。

当访问一个对象的属性（方法）时，首先从对象本身查找，找到则返回该属性；若未找到，则继续查找其原型对象的属性，如果还找不到，则会沿着原型链一直向上查找，一直到根。

因此，原型对象的用途就是就是为每个实例对象存储共享的方法和属性

二、继承

既然原型对象能为每个实例对象存储共享的方法和属性，我们就可以通过原型对象来实现继承

1、原型链继承

通过原型链实现继承关系，具体来说，就是将父类的实例作为子类的原型

function parent(name) {
  this.name = name || 'father'
  this.getName = function () {
    console.log('name', this.name)
  }
  this.feature = []
}

parent.prototype.action = function () {
  console.log('action')
}

function child() {}
child.prototype = new parent('son')
child.getName() // child.getName is not a function
child.action() // 'action'

特点：

  • 纯粹的继承关系，实例是子类的实例，也是父类的实例
  • 父类新增的原型方法或属性，子类都能访问
  • 简单易实现

缺点：

  • 来自父类对象的引用属性是所有子类实例共享的，如feature就是所有实例共享
  • 创建子类实例时，无法向父类构造函数传参
  • 无法实现实例多继承
